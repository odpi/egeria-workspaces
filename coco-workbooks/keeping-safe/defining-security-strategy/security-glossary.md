# Security glossary

This file include the commands to create a glossary of security terms.
This glossary supports the development of a security strategy.

It includes the following glossary terms:

* Security
* STRIDE
* api, 
* vulnerability, 
* mitigation
* threat modelling, threat landscape, threat model
* security controls,
* risk assessment

# Create Glossary Term

## Display Name

Security

## Glossary Name

Glossary::Security-Glossary

## Summary

Security is a state of being in which a person, organization, location, or system is protected against potential harm.

## Description

The focus of security (such as a person, organization, location, or IT system) is called the **security referent**.  The security referent has a relationship with their environment, sometimes called their **security context**.  This may be beneficial, or hostile (or most likely a mixture of both).   The security referent has **security capabilities** that they can use to protect themselves from the hostile forces in their environment.

The **security strategy** lays out how the referent's security capabilities are used to protect themselves.

## URL

https://en.wikipedia.org/wiki/Security

## Abbreviation

Sec

## Usage

The term is used to both describe the state of being secure, and the activities, defences and tools used to protect against security threats.

# Create Glossary Term

## Display Name

Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege (STRIDE)

## Glossary Name

Glossary::Security-Glossary

## Summary

STRIDE is an acronym describing six categories of security threats to an IT system.

## Description

The terms that make up the stride model are:

* Spoofing - a spoofing attack is a situation in which a person or program successfully identifies as another by falsifying data, to gain an illegitimate advantage.
* Tampering - tampering is the act of modifying data or software in a way that is not authorized.
* Repudiation - repudiation is the act of denying having performed an action, such as a transaction or a message.
* Information Disclosure - information disclosure is the act of revealing sensitive information to unauthorized parties.
* Denial of Service - denial of service is the act of preventing legitimate users from accessing a system or service.
* Elevation of Privilege - elevation of privilege is the act of gaining unauthorized access to a system or resource with higher privileges than intended.

## URL

https://en.wikipedia.org/wiki/STRIDE_model

## Abbreviation

STRIDE

## Usage

STRIDE is used as a checklist of security threats to consider with respect to a system component during threat modelling.
